### HTML

1. 나만의 갤러리에서 사진 9장은 HTML 태그 중 어떤 요소로 표현하는 것이 좋을까? 이유는 뭐라고 생각했는가?
    - `<img>` 태그를 사용했습니다. 다른 태그에서도 src를 통해서 image를 불러올 수 있겠으나 “이미지”를 표현하고 다른 개발자가 이해하기에 가장 적합한 태그라고 생각했습니다.
2. 뼈대 코드를 기반으로 이미지를 로드하는 방식은 크게 2가지로 나뉘는데 무엇이고 어떻게 사용하고 있는가?
    - 네트워크를 통해 이미지를 받아오는 것과 로컬에 존재하는 이미지를 가져오는 것이라고 생각합니다. 저는 저장소에 이미지를 가져왔고 상대경로를 사용하여 이미지를 로드했습니다.
3. 시맨틱 태그(예: **`<header>`**, **`<main>`**, **`<section>`**, **`<footer>`** 등)로 문서를 구분할 때, 어떤 기준으로 섹션을 나누고 태그를 적용하는 것이 좋을까?
    - 의미와 역할을 기준으로 나누는게 좋다고 생각합니다. 아래는 질문의 예시에 해당하는 시맨틱 태그에 대한 의미이며 해당 기준으로 요소를 배치하는 게 좋다고 생각합니다.
        - `<header>` :  페이지의 소개, 로고, 내비게이션
        - `<main>` :  페이지의 핵심 콘텐츠
        - `<section>` : 주제별로 묶인 콘텐츠
        - `<footer>` : 저작권, 연락처 등 마무리 정보
4. **`<img>`** 태그의 **`alt`** 속성은 왜 필요하며, 어떻게 작성하는 것이 좋을까?
    - 안드로이드에도 Image를 사용하려면 contentDescription이라는 패러미터를 작성하게끔 되어있습니다. 이미지를 직접 구분하기 어려운 분들을 위해 작성하고 리더기가 읽어서 도움을 줄 수 있기에 이미지에 대한 간략한 설명을 작성하는 게 좋다고 생각합니다.
5. 이미지를 로드할 때, 상대 경로(relative path)와 절대 경로(absolute path)를 혼용하면 어떤 문제가 발생할 수 있을까?
    - 로컬에서는 괜찮았던 절대 경로가 운영 환경에서는 이상해진다거나 하는 문제가 있을 것 같습니다. 그렇기에 상대 경로로 통일하는 것이 좋다고 생각합니다.
6. **`<meta>`** 태그의 viewport 설정과 Open Graph 태그는 각각 어떤 역할을 하는가?
- 모바일 브라우저는 기본적으로 데스크탑 페이지라고 가정해서 화면을 축소해서 보여주는데, viewport 설정을 통해서 모바일 화면을 따로 설정해서 보여줄 수 있습니다.
- Open Graph 태그는 SNS나 플랫폼에 공유시 미리보기를 제공함으로 다른 사용자에게 흥미를 불러일으킬 수 있고 링크만 전달하는 것 보다 더 많은 정보를 전달할 수 있습니다.

### CSS

1. 나만의 갤러리에서 9장의 이미지를 보기 좋게 배치하려면, Flexbox와 CSS Grid 중 어떤 것이 더 적합할까? 그 이유는?
    - Flexbox는 가로나 세로 하나의 방향으로 이미지를 배치할 수 있습니다. 만약 이미지를 스크롤 가능하게 보는걸 원한다면 올바른 선택이겠으나 한번에 여러 이미지를 보기에는 CSS Grid가 더 좋다고 생각합니다.
2. 태그 선택자, 클래스 선택자, ID 선택자는 각각 언제 사용하는 것이 적절할까?
    - 태그 > 클래스 > ID 순으로 더 넓은 적용 범위를 갖습니다. ID 선택자는 하나의 컴포넌트에 대해서만 적용 가능하기에 특별한 스타일을 정의할 때 유용하다고 생각합니다. 클래스 선택자는 하나 이상의 컴포넌트이지만 페이지를 넘어서는 전역 범위가 아닐 때 유용합니다. 마지막으로, 태그 선택자는 클래스를 추가하거나 ID를 추가할 필요가 없기에 전역으로 설정이 필요할 때 유용할 것 같습니다.
3. CSS를 HTML 내에 직접 작성하는 것과 별도 파일로 분리하는 것의 차이는 무엇일까?
    - HTML내에 직접 작성하는 건 다른 페이지에서 재사용하기 어렵고 파일로 분리하면 재사용성이 올라가는 장점이 있습니다.


### JavaScript

1. **`document.getElementById()`**와 **`document.querySelector()`**의 차이는 무엇인가?
    - getElementById는 Id 선택자만 사용해서 요소를 가져올 수 있는 반면 querySelector는 CSS 선택자, 클래스 선택자, 속성 선택자도 이용 가능해 더 유연한 옵션을 제공합니다.
2. 이벤트 리스너를 사용해서 "어떤 일이 일어났을 때 어떤 동작을 실행"하게 만들 수 있다. 자기소개 페이지에서 활용할 수 있는 이벤트에는 어떤 것들이 있을까?
    - 현재 페이지에서 활용할 수 있는 이벤트 및 확장 할 수 있는 기능은 다음과 같습니다.
        - 깃허브 프로필 이동
            - 단순한 링크 이동이지만 이벤트 리스너를 통해서 링크 이동 + 사용자 추적까지 겸할 수 있습니다.
        - TIL 등록
            - DB 혹은 로컬 스토리지에 저장하기
        - 갤러리 이미지
            - 이미지를 클릭하여 설명 붙이기
3. **`innerHTML`**로 DOM을 조작할 때 주의해야 할 점은 무엇일까?
    - 현재는 단순하게 입력값을 받고 있지만 alert(’message’)를 입력하면 실제로 코드가 실행되는 문제가 있습니다. 이를 Cross Site Scripting이라 하기에 이 점을 주의할 필요가 있습니다.

### 배포

1. GitHub Pages로 배포한 페이지에서 TIL을 작성하면, 새로고침하거나 재배포했을 때 데이터가 유지될까? 그 이유는?
    - 아니요, 지금의 페이지에서 구현한 바는 TIL을 작성해도 로컬이나 DB에 저장하는 방식이 아니기에 저장되지 않습니다. 현재는 단순히 브라우저 메모리에만 존재하므로, 새로고침하거나 재배포했을 시 모든 데이터가 사라지고 초기화됩니다.
2. JavaScript로 추가한 데이터를 영구적으로 저장하려면 어떤 방법이 있을까?
    - DB에 저장해서 서버와 통신해서 가져오기, 로컬 스토리지에 저장하기가 떠오릅니다. 단, 로컬 스토리지에 저장하는 방식은 브라우저에서 사이트 데이터 삭제를 하면 정보가 없어지기에 DB에 저장해서 서버와 통신해서 가져오는 방식이 더 영구적으로 저장하는 방법에 가깝다고 생각합니다.
